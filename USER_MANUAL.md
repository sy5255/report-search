# 📖 불량 분석 RAG 시스템 (Report Search) 사용 설명서

안녕하세요! **불량 분석 RAG 시스템**에 오신 것을 환영합니다. 
본 시스템은 사내 기술 문서(RAG)의 지식 검색과 불량 데이터베이스(DB)의 통계 조회를 채팅 한 번으로 손쉽게 활용할 수 있도록 돕는 지능형 대화(AI) 비서입니다.

처음 사용하시는 분들도 쉽게 시스템의 모든 기능을 100% 활용할 수 있도록, 웹 화면의 구조와 주요 기능의 사용 방법들을 자세히 안내해 드립니다.

---

## 1. 시작하기 (로그인 및 기본 화면)

1. **로그인**: 발급받으신 사내 계정 ID와 비밀번호를 입력하여 시스템에 접속합니다.
2. **채팅 화면 진입**: 접속하시면 `New Chat` 이라는 새로운 대화창이 시작됩니다. 좌측에는 **과거 대화 내역(Sessions)**이 위치하며, 중앙에는 AI 비서와 대화할 수 있는 커다란 **채팅창**이 위치해 있습니다.
3. **전문 지식 범위(Index) 선택**: 검색 전, 시스템 설정(주로 우측 하단이나 설정 메뉴)에서 검색 대상 인덱스(예: `rp-ifa1-ver1-full` 등)를 선택할 수 있습니다. 시스템이 기본적으로 가장 적절한 모드를 잡고 있으니, 특별한 경우가 아니면 기본값을 사용하셔도 무방합니다.

---

## 2. 💡 핵심 기능 1: 전문 기술 문서 검색 (RAG Knowledge)

이 기능은 `"어떤 불량의 원인이 무엇인가요?"` 혹은 `"특정 공정에 대한 해결책을 알려주세요"` 같은 종류의 질문을 할 때 활용합니다. 방대한 사내 보고서와 메일에서 가장 정확한 원리를 찾아 요약해 줍니다.

**✅ 사용 방법 및 예시**
*   **질문하기**: 채팅창에 자연스럽게 아래처럼 질문해 보세요.
    *   *"최근 발생한 파티클 불량의 주요 발생 원인과 해결 가이드를 찾아줘."*
    *   *"SF2 공정노드에서 발생하는 패턴 무너짐 관련 보고서 요약해 줘."*
*   **결과 확인**:
    *   AI가 사내 문서들을 검색한 뒤, 이해하기 쉬운 **카드형 포맷(인용구 `>`)**으로 내용을 요약하여 답변해 줍니다.
    *   답변 중간에 달린 **출처 번호(예: [1], [2])**에 마우스를 올리면 어떤 사내 웹 문서/보고서에 기반하여 답변했는지 신뢰성 있는 **원문(근거) 출처**를 확인할 수 있습니다.

---

## 3. 📊 핵심 기능 2: DB 통계 및 수치 분석 (DB Analysis)

말장난이나 긴 글 대신, `"최근 가장 많이 터진 불량은?"`, `"특정 설비의 에러 건수는?"` 처럼 구체적인 **데이터 수치나 순위 테이블**이 필요할 때 사용하는 기능입니다.

**✅ 사용 방법 및 예시**
*   **질문하기**: 원하는 통계 조건을 채팅창에 명확히 입력해 보세요.
    *   *"최근 3개월 동안 가장 많이 발생한 불량명 순위 5개 뽑아줘."*
    *   *"PC 모듈에서 발생한 불량 리스트를 정리해 줘."*
*   **결과 확인**:
    *   AI가 텍스트 대신 깔끔한 **표(Table)** 형태로 통계/목록 데이터를 정리하여 보여줍니다.
    *   결과 안심 확인: 데이터 표 제일 하단에 있는 **`💡 실행된 최종 SQL 쿼리 보기`** 토글을 클릭하면, AI가 DB에서 값을 가져오기 위해 직접 실행한 정확한 쿼리문을 눈으로 확인할 수 있습니다.

---

## 4. 🧠 하이브리드 검색 및 "추천 액션(Action Chips)" 활용

본 AI 시스템의 가장 큰 장점은 문서 검색과 통계 분석이 **동시에 상호작용** 한다는 점입니다. 

**✅ 복합 질문 던지기 (Hybrid)**
*   *"최근 1개월간 발생한 빈도수 1위 불량 이름이 뭔지 통계를 내고, 그 불량이 왜 발생하는지 기술 원인도 같이 알려줘."*
*   이렇게 질문하면 AI 스스로 알아서 1️⃣ **DB 도구**를 써서 표를 그리고, 2️⃣ **문서 검색 도구**를 써서 원인을 요약하여 한 번에 보여줍니다.

**✅ 추천 액션 칩 (UI 버튼 클릭) 활용**
채팅을 주고받다 보면 AI가 답변 바로 밑에 마우스로 클릭할 수 있는 **[액션 버튼]**을 자동으로 띄워줍니다.
*   예를 들어, 통계 질문을 했는데 결과가 마음에 안 들 경우: **`[⚠️ 조건을 넓혀서 다시 검색]`** 버튼이 생성됩니다. 이 버튼을 클릭하면 AI가 알아서 검색 범위를 넓혀 DB를 다시 조회합니다.
*   표를 보고 나서 원인이 궁금해졌을 경우: 타자를 칠 필요 없이 **`[📖 문서 검색 Agent 호출하기]`** 버튼을 누르면 AI가 현재 흐름을 기억한 채 바로 기술 문서를 검색해 줍니다.

---

## 5. 부가 기능 안내

*   **대화 이어가기(Context 기억)**
    *   채팅 중 계속해서 "그럼 그 두 번째 결과에 대해서 더 자세히 말해봐" 처럼 **대명사를 사용**해도 AI가 이전 대화 맥락을 기억하고 찰떡같이 이해합니다. (단, 너무 오래전 맥락보다 **최근 3번의 대화**를 가장 잘 기억합니다)
*   **세션 내역 관리 (Sessions)**
    *   화면 왼쪽의 목록에서 과거에 대화했던 채팅 방을 다시 클릭하면 옛날 분석 내역을 언제든지 다시 열어보고 이어서 대화할 수 있습니다.
*   **분석 모드 직접 제어 (팁)**
    *   가끔 AI가 의도를 잘못 파악한다면, 문장 맨 앞에 강제 태그를 붙여보세요!
    *   `[DB_ANALYSIS] 최근 3개월 에러 순위 알려줘` -> (무조건 DB 분석 모드로 직행)
    *   `[RAG_KNOWLEDGE] 이 공정의 원리가 뭐야?` -> (무조건 문서 검색 모드로 직행)
